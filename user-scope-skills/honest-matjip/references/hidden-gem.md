# Hidden Gem Mode

사용자가 "숨은", "우리만 아는", "히든" 같은 표현을 쓸 때 적용되는 **역방향 필터**.
기본 모드는 리뷰 수가 많은 곳을 선호하지만, Hidden Gem 모드는 **리뷰 수가 적은데
리뷰어 이력은 탄탄한 곳**을 적극 찾는다.

## 언제 활성화되는가

사용자 입력에 다음 중 하나가 있으면 자동 진입:

**강한 트리거 (단독으로 활성화):**
- "숨은", "숨겨진", "히든", "hidden"
- "우리만 아는", "둘만 아는", "아무도 모르는"
- "아지트", "골목", "덜 알려진"
- "바이럴 안 된", "소문 안 난", "인스타 안 탄"

**약한 트리거 (다른 맥락과 결합 시):**
- "조용한", "아는 사람만", "작은", "좁은"
- "사람 없는", "한적한"

**거꾸로 트리거 안 됨:**
- "유명한", "핫한", "1등", "인기", "맛집 베스트"
- "검색 1위", "SNS 유명"

## 필터 공식 (기본 모드 대비)

| 조건 | Standard | Hidden Gem |
|------|----------|-----------|
| 리뷰 수 범위 | 50~∞ 선호 | **5~40 타겟**, 100+ 적극 배제 |
| 신뢰 리뷰어 최소 | 3명 | **2명** (완화) |
| 종합 신뢰 점수 컷 | 0.5 | **0.45** (완화) |
| 리뷰어 평균 최소 | - | **200건 이상 필수** (강화) |
| 후보 수 | 5~8개 | **3~5개** (소수 정예) |

**핵심 역설:** 리뷰 수 샘플이 작은 대신 **각 리뷰어의 이력 깊이**로 신뢰 보상.
리뷰 10건이지만 그 리뷰어들이 평균 400+ 리뷰 Local Guide면 "잘 아는 사람들이 단골"인 증거.

## 추가 시그널 — Hidden Gem 전용 가점

### 1. 컬트 언어 감지 (Cult Language Signal)

진짜 단골만 쓰는 감정적 표현. 광고 계정은 절대 못 쓰는 톤.

**강한 컬트 시그널 (발견 시 +0.2):**
- "인생에서 처음 리뷰를 써봅니다"
- "서울 안 가도 된다" / "강남 안 가도 된다" (서울 기준 비교 우위)
- "사장님 그대로 있어주세요"
- "단골이 되어버렸" / "무조건 단골예약"
- "한번만 믿고 가보세요"
- "이제 다른 곳은 못 가겠어요"
- "사람들한테 알려주기 싫은" (역설적 극찬)
- "대방문의사 500%" 같은 극단적 재방문 의지

**중간 컬트 시그널 (발견 시 +0.1):**
- "아지트 같은", "조용한 동네"
- "혼자 알고 싶은"
- "단골이 많은 집"
- "여기보다 싸고 좋은 곳 없다"

### 2. 카테고리 희소성 (Category Rarity Signal)

해당 지역에서 카테고리 자체가 드문 경우 +0.15:
- 예: 광교에서 "LP 컨셉 바", "정통 스페인 요리", "클래식 칵테일 바"
- 예: 행궁동에서 "모던 프렌치", "와인 페어링 전문점"

검증 방법: 동일 지역에 같은 카테고리로 검색했을 때 비슷한 후보가 3개 미만이면 희소.

### 3. 위치 숨음도 (Location Hiddenness Signal)

물리적으로 찾기 어려운 위치 +0.1:
- 아파트 단지 내 상가 (특히 지하)
- 오피스빌딩 지하 또는 3층 이상
- 골목 안쪽, 간판 약한 곳
- 도로변 아니고 안쪽 블록

검증 방법: Google Maps 주소에 "지하", "B1", "2층", "3층", "아파트", "단지", "상가" 포함 여부.

## 수정된 점수 공식

```
restaurant_trust_hidden =
    0.40 * mean(reviewer_trust for reviewer in analyzed_reviewers)
  + 0.20 * min(trusted_reviewer_count / 5, 1.0)  # 5명으로 만점 (완화)
  + 0.15 * min(reviewer_avg_reviews / 400, 1.0)  # 리뷰어 슈퍼 유저 가점
  + bonus_cult_language                           # 0 ~ 0.2
  + bonus_category_rarity                         # 0 or 0.15
  + bonus_location_hidden                         # 0 or 0.1

# 최대 1.0 cap
restaurant_trust_hidden = min(restaurant_trust_hidden, 1.0)
```

## 필수 검증 단계 (Hidden Gem 추천 전에)

Standard 모드는 리뷰 수로 "대중 검증"을 확보하지만, Hidden Gem 모드는 리뷰 수가 적으므로
**다음 4가지 중 최소 2가지**가 충족되어야 추천 가능:

1. ✅ 리뷰어 평균 200건 이상 (Local Guide 슈퍼 유저 단골)
2. ✅ 컬트 언어 1건 이상 감지
3. ✅ 카테고리/위치 희소성 있음
4. ✅ 체험단/협찬 키워드 0건

2가지 미만이면 **"숨어 있긴 하지만 검증 부족으로 제외"** 표기.

## 제외 강화 규칙

Hidden Gem 모드에서도 다음은 **무조건 제외**:

- 리뷰 수 1~4건 (표본 극소 + 컬트 시그널 없음)
- 5.0★ 만점인데 리뷰어가 모두 1~5건짜리 신생 계정 (광고 의심)
- 체험단/협찬 키워드 1건이라도 감지
- 프로필 없는 리뷰어만 있음

## 출력 포맷 조정

Hidden Gem 모드 출력에는 **다음 요소가 필수**:

1. **"숨은 이유" 섹션** — 왜 이 곳이 숨어 있는지 (위치·카테고리 희소성·리뷰 수)
2. **"컬트 리뷰 인용"** — 감정 단골 표현을 적어도 1개 원문으로 인용
3. **"표본 작음 경고"** — "리뷰 10건이라 표본은 작지만, 이력이 탄탄한 리뷰어들의 평가"
   같은 솔직한 고지
4. **"실패 시 책임 한계"** — "숨은 곳 특성상 기대치 조정 필요. 취향 안 맞을 가능성 존재"

## 주의사항

- Hidden Gem 모드는 **기대치가 낮은 사용자**에게만 제안해야 한다. "완벽한 저녁"을
  원하는 사용자에게 리뷰 10건짜리 곳을 추천하면 리스크가 크다.
- 사용자가 "중요한 날", "기념일", "프로포즈" 등을 명시하면 Hidden Gem 모드를
  비활성화하고 Standard 모드로 복귀 (안전 우선).
- **"우리만 아는"이라는 표현은 대부분 놀이·호기심 컨텍스트**이므로, 결혼기념일 같은
  안전 필요 맥락과는 다르다. 맥락 판단이 우선.
