# Output Format

## 기본 구조

```markdown
# [분위기/상황] 맛집 추천

## 데이터 소스 접근 현황
- Google Maps: [성공/실패]
- 카카오맵: [성공/실패]
- 네이버 플레이스: [성공/차단됨/미시도]

---

## [지역명 1]

### 1. [맛집 이름]
- **위치:** [주소 또는 동네]
- **카테고리:** [한식/이탈리안/일식 등]
- **가격대:** [1인 기준 대략]
- **신뢰 요약:** 최근 리뷰어 [N]명 분석 → [M]명 신뢰 (신뢰율 XX%)
  - 다양한 장소 리뷰 이력 보유: [N]명
  - 음식점 전문 계정 (의심): [N]명
  - 체험단/협찬 표기 감지: [N]건
- **신뢰된 리뷰 인용:**
  > "[실제 리뷰 텍스트]" — [리뷰어 프로필 요약: 리뷰 XX건, 다양한 장소]
- **경고:** [해당 시 — "체험단 리뷰 2건 감지" 등]

### 2. [맛집 이름]
...

---

## [지역명 2] (있을 경우)
...

---

## 제외된 후보 (투명성)
| 맛집 | 제외 이유 |
|------|----------|
| [이름] | 신뢰 리뷰어 2명 미만 (기준: 3명) |
| [이름] | 음식점 전문 리뷰어 비율 80% — 광고 의심 |

---

## 분석 한계
- [해당 시] 네이버 플레이스 접근 차단으로 Google Maps/카카오맵 데이터만 사용
- [해당 시] 리뷰어 프로필 표본 부족 (X개 맛집에서 3명 미만)
```

## 필수 포함 요소

1. **데이터 소스 접근 현황** — 어떤 소스가 작동했고 어떤 소스가 실패했는지
2. **리뷰어 분석 근거** — 각 맛집마다 리뷰어 프로필 분석 결과 요약
3. **신뢰된 리뷰 인용** — 신뢰된 리뷰어의 실제 리뷰 1~2개
4. **제외된 후보** — 분석했지만 탈락한 맛집과 사유 (투명성)
5. **분석 한계** — 기술적 제한이 있었으면 솔직히 명시

## 금지 요소

- LLM 지식 기반 "여기 맛있어요" 추천 (근거 없음)
- 점수를 절대적으로 해석하는 표현 ("안전합니다", "보장합니다")
- 크롤링 실패를 숨기는 것 (항상 명시)
- 분석하지 않은 맛집을 추천하는 것

## 짧은 버전 (사용자가 간결한 답을 원할 때)

```markdown
## [지역] 추천 3곳

1. **[맛집명]** — [카테고리], [가격대]
   신뢰도: 리뷰어 8명 중 6명 신뢰 | 체험단 0건
   > "[리뷰 인용]"

2. **[맛집명]** — ...

3. **[맛집명]** — ...

*분석 기반: Google Maps + 카카오맵 | 네이버 차단됨*
```
