# Reviewer Profile Analysis

리뷰어 한 명의 프로필을 방문해서 "이 사람은 일반 사용자인가, 광고성 계정인가?"를 판단하는 휴리스틱.

## 접근 경로

### Google Maps (Primary)
- URL: `https://www.google.com/maps/contrib/[USER_ID]/reviews`
- 공개 페이지, 로그인 불필요
- chromux로 접근 → 페이지 내 정보 추출

**추출 가능 필드:**
- 총 리뷰 수
- Local Guide 레벨 (1~10)
- 기여 유형 분포 (리뷰, 사진, 업데이트, Q&A)
- 최근 리뷰 목록 (장소 유형별 다양성 판단)

### 카카오맵 (Secondary)
- 리뷰어 프로필 URL: 미문서화, 리버스엔지니어링 필요
- place 상세 페이지에서 리뷰어 이름 클릭 → 프로필 이동 여부 확인
- 실패 시: 리뷰어 이름 + 리뷰 텍스트만으로 부분 판단

**추출 시도:**
- 리뷰 수
- 리뷰 장소 유형 목록 (음식점 비율)
- 단골 표기 여부

### 네이버 플레이스 (Tertiary, Degrade 허용)
- anti-bot 강도: 높음 (DataDome + 행동 분석)
- 리뷰어 프로필 경로: MY플레이스 (본인 전용일 수 있음)
- 블로그 연동 리뷰: 블로거 프로필로 우회 가능

**크롤링 실패 대응:**
1. 첫 시도 실패 → 2~3초 대기 후 재시도 1회
2. 재시도 실패 → 네이버 전체를 이 세션에서 포기, Google + 카카오만으로 진행
3. "네이버 접근 차단으로 미분석" 명시

## 신뢰 신호 (Signals)

| 신호 | 신뢰 | 의심 | 가중치 |
|------|------|------|--------|
| 음식점 전문 비율 | `< 0.3` (다양한 장소 리뷰) | `> 0.7` (음식점만) | 30% |
| 총 리뷰 수 | `> 50` | `< 5` (신생 계정) | 20% |
| 체험단/협찬 키워드 | 없음 | `#협찬`, `체험단`, `제공받아`, `광고` 포함 | 25% |
| 계정 연차 | `> 180일` | `< 30일` (최근 생성) | 15% |
| 프로필 풍부도 | 이미지 있음 + 바이오 있음 | 빈 프로필 | 10% |

## 계산 공식

```
reviewer_trust =
    0.30 * (1 - restaurant_only_ratio)
  + 0.20 * min(total_reviews / 50, 1.0)
  + 0.25 * (0.0 if has_sponsored_keyword else 1.0)
  + 0.15 * (0.0 if account_age_days < 30 else 1.0)
  + 0.10 * profile_richness   # 0.0 ~ 1.0
```

- `restaurant_only_ratio` = 음식점 리뷰 수 / 전체 리뷰 수
- `profile_richness` = 프로필 이미지(0.5) + 바이오(0.5) 합산
- `has_sponsored_keyword` = 해당 맛집 리뷰 또는 최근 리뷰 10개 내 키워드 감지

## 체험단/협찬 키워드 목록

**확정 키워드 (Strong signal):**
- "체험단", "협찬", "제공받았", "광고", "#ad", "무료로 제공"
- "소정의 원고료", "방문 리뷰", "인플루언서 초대"

**약한 키워드 (Weak signal — 단독으로 판단하지 않음):**
- "제공", "초대", "이벤트" (일반적 맥락에서도 사용됨)
- 이런 키워드는 다른 의심 신호와 **겹칠 때만** 감점

## 분석 최소 기준

- 후보 맛집당 **최소 5명** 리뷰어 프로필 분석 시도
- 접근 가능 리뷰어가 3명 미만이면 "표본 부족" 명시
- 리뷰어 프로필 접근 실패는 "미분석"으로 기록 (추정 점수 금지)
