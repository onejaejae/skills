# clue-research-ai 도메인 Knowledge

## 제품 요약

HRS(Healthcare data Research Suite)의 **AI 연구지원** 기능. 카카오헬스케어가 제공하는 의료 데이터 기반 연구 플랫폼의 핵심 모듈.

연구자가 자연어로 연구 주제를 입력하면, AI가 임상 연구설계의 전체 구조를 자동 생성하고, CDW(병원 데이터)에서 실제 유효성을 검증하는 기능.

**핵심 사용자:** 임상연구자 (의사, 연구원) — 특히 연구 경험이 적은 초기 연구자
**핵심 가치:** 수주~수개월 걸리던 연구설계 과정을 분 단위로 단축

**제품 계층:**
```
HRS 통합홈
├── UDM 연구검색
│   ├── AI 연구지원 ★ (이 제품)
│   ├── 연구코호트 (환자 집단 정의)
│   ├── 데이터셋 구성
│   ├── 통계분석 / 기계학습
│   ├── 코드 통합탐색 / 코드set
│   └── ...
└── 지표모니터
```

## 핵심 사용 플로우

```
[Step 1] 연구 주제 입력 (자연어)
    "SGLT-2 억제제가 심혈관 사건을 줄일까?"
        ↓
[Step 2] AI 연구설계 자동 생성 (7개 섹션)
    가설 → 연구목적 → 방법론 → 코호트 → 변수 → SAP → 키워드
        ↓
[Step 3] 연구자 검토 & 수정
    직접 수정 또는 AI 대화로 수정 (문답모드/화면적용 모드)
        ↓
[Step 4] 데이터 검증
    키워드별 CDW 코드 조회 → 환자수/이벤트수 확인 → 유효 코드 선별
        ↓
[Step 5] 코드set 저장 & 활용
    유효 코드 묶음 → 연구코호트에서 재사용
```

AI가 자동화하는 임상연구 단계: [2]가설 ~ [6]데이터 수집

## 도메인 용어집

### 임상 연구 핵심 용어

| 용어 | 정의 | 제품에서의 역할 |
|------|------|---------------|
| 연구설계 (Study Design) | 임상 연구의 전체 구조 정의 | AI가 7개 섹션으로 자동 생성 |
| 가설 (Hypothesis) | 검증하고자 하는 명제 | AI 생성 → 연구자 검토 |
| 연구코호트 (Cohort) | 연구 대상 환자 집단 (포함/제외 기준) | Inclusion/Exclusion 조건으로 정의 |
| 변수 정의 (Variable Definition) | 측정 항목의 조작적 정의 | 독립변수/종속변수/교란변수/공변량 |
| Index Date | 각 환자의 관찰 시작 기준일 | 모든 변수가 이 시점 기준으로 계산 |
| SAP (Statistical Analysis Plan) | 통계 분석 계획 | Cox 모델, PSM, Kaplan-Meier 등 |
| 키워드 | 의학 개념→코드 매핑용 용어 | 4개 도메인(진단/약물/검사/수술)으로 분류 |
| 코드set | 특정 의학 개념의 코드 묶음 | 검증된 코드 저장 → 재사용 가능 |
| CDW (Clinical Data Warehouse) | 병원 진료/처방/검사 데이터 저장소 | 데이터 검증의 데이터 소스 |
| 데이터 검증 | 키워드가 CDW에 존재하는지 확인 | 환자수/이벤트수로 유효성 판단 |

### 의학 코드 체계

| 코드 체계 | 대상 도메인 | 예시 |
|-----------|-----------|------|
| ICD-10 | 진단 | E11 = 제2형 당뇨, I21 = 심근경색 |
| ICD-09 | 진단 (과거 데이터) | |
| SNOMED CT | 전체 | 240349003 = Cholera due to Vibrio cholerae |
| KCDPCM | 진단 (한국) | 한국표준질병사인분류 |
| ATC | 약물 | A10BK01 = Dapagliflozin |
| RxNorm | 약물 | 성분명 기반 |
| LOINC | 검사 | 2345-7 = 혈당검사 |
| ICD-PCS | 수술/시술 | 02100Z9 = 관상동맥우회술 |
| UDM Concept ID | 전체 | HRS 자체 통합 코드 |

### 변수 종류

| 종류 | 역할 | 예시 (SGLT-2 연구) |
|------|------|-------------------|
| 독립변수 (Exposure) | 원인 | SGLT-2 억제제 복용 여부 |
| 종속변수 (Outcome) | 결과 | MACE 발생 여부 |
| 교란변수 (Confounder) | 결과에 영향주는 요인 | 나이, 성별, BMI |
| 공변량 (Covariate) | 분석에서 보정하는 변수 | 고혈압, 이전 심혈관 이력 |

## 비즈니스 규칙

### AI 대화 모드
- **문답 모드**: AI와 자유 대화 (화면에 영향 없음)
- **화면 적용 모드**: AI 답변이 연구설계 폼에 직접 반영

### 데이터 검증 판단 기준
1. 코드가 CDW에 **존재하는가?** (Valid vs N/A)
2. **환자수가 충분한가?** (통계적 검정력)
3. 코드가 **연구 의도에 정확히 맞는가?**

### 코드set 규칙
- 같은 도메인의 코드만 하나의 코드set으로 묶을 수 있음
- 저장된 코드set은 다른 연구에서 재사용 가능
- 다른 연구자가 공유한 코드set 불러오기 가능

### 제품 위치
- **후향적 관찰 연구** 지원 (이미 쌓인 병원 데이터 활용)
- 전향적 연구(미래 관찰)는 대상 아님

## 데이터 구조 (엔티티 관계)

```
User 1──N AI연구지원(Session)
AI연구지원 1──1 연구설계(ResearchDesign)
AI연구지원 1──1 데이터검증(DataValidation)
연구설계 1──N 연구코호트조건(CohortCriteria)
연구설계 1──N 변수정의(VariableDefinition)
연구설계 1──N 키워드(Keyword)
키워드 N──1 도메인(Domain: 진단/약물/검사/수술·시술 + 7개 추가 도메인)
키워드 1──N 검증코드(ValidationCode)
검증코드 1──1 상태(Status: Valid/N/A)
검증코드 N──N 코드set(CodeSet)
코드set ──→ 코드통합탐색 / 연구코호트 (재사용)
AI연구지원 N──1 프로젝트(Project, 선택적)
AI연구지원 N──1 공간(내연구/프로젝트)
```

## 연관 시스템

| 시스템 | 역할 | 연동 방식 |
|--------|------|----------|
| CDW | 병원 데이터 소스 (진료/처방/검사) | UDM Meta 테이블 (읽기 전용) |
| 연구코호트 | 환자 집단 구성 | 코드set 기반 환자 검색 |
| 코드 통합탐색 | 코드set 관리 | 코드set 저장/조회 |
| clue-login | 사용자 인증 | JWT 기반 |
| 코호트 생성기 | 외부 코호트 생성 API | REST API |

## 알려진 Pain Points

- 연구 경험이 적은 초기 연구자: 가설→코호트→변수→SAP 과정이 수주~수개월 소요
- 키워드→코드 매핑: ICD-10 코드북 수작업 탐색 필요
- 데이터 검증: 전통적으로 데이터팀 요청 후 수일~수주 대기
- 코드 재사용: 개인 엑셀/노트 관리, 공유 불가
