# stepback

원샷 관점 리셋 스킬 — 현재 작업을 스캔해 추상적 재구성 질문 1개와 3가지 빠른 점검(범위 이탈·사이드 이펙트·더 나은 접근법)을 10줄 이내로 즉시 제공합니다.

## 설치

```bash
/plugin install stepback@ai-registry
```

## 사용법

### 트리거

- `/stepback`, `step back`, `한발 물러서`, `넓은 관점`
- `놓치는 거 없나`, `방향 맞나`, `zoom out`, `큰 그림`
- `blind spot`, `재확인`, `잠깐 멈춰`

### 출력 형식

```
**Step-Back:** [현재 구체적 작업을 추상 원칙으로 재구성한 질문]

**Scope Drift:** [원래 목표에서 이탈 여부 — 1문장]
**Side Effects:** [변경이 다른 곳에 미칠 영향 — 1문장]
**Better Approach:** [더 근본적인 해결책 존재 여부 — 1문장]

Carry on.
```

### Step-Back 질문 패턴 (DeepMind step-back)

| 현재 작업 | Step-Back 질문 예시 |
|----------|------------------|
| null 포인터 버그 수정 | 이 모듈의 에러 처리 전략 자체가 올바른가? |
| API 엔드포인트 추가 | 이 기능이 API 레이어에 있어야 하는가? |
| 함수 리팩토링 | 이 함수가 존재해야 하는가, 호출부가 문제인가? |
| 쿼리 최적화 | 이 시점에 이 쿼리를 호출해야 하는가? |

### 특징

- **원샷** — 대화 없이 즉시 반환, 후속 질문 없음
- **10줄 이내** — 간결함이 핵심
- AskUserQuestion, 에이전트 스폰, 코드 생성 없음
- No-context fallback 처리 (컨텍스트 없을 때 경고 후 일반 패턴으로 분석)

### 유사 스킬 비교

| 스킬 | 목적 | 선택 기준 |
|------|------|----------|
| `/stepback` | 작업 중 원샷 방향 점검 | 뭔가 잘못된 것 같을 때 즉시 확인 |
| `/discuss` | 소크라테스식 아이디어 탐구 | 시작 전 개념 탐구 |
| `/mirror` | 요청 이해 검증 | Claude가 내 요청을 이해했는지 확인 |
| `/scope` | 특정 변경의 영향 범위 분석 | 구체적 변경안이 있을 때 |
| `/tribunal` | 고위험 결정의 다관점 리뷰 | 아키텍처 의사결정 |
