# Phase 3: Product Pipeline (상세 가이드)

**RULE: 화면이 아니라 유저의 업무 흐름을 따라가라.**
- X: "Screen A에서 버튼 클릭 → Screen B"
- O: "연구자는 먼저 주제를 입력하고, AI가 설계를 생성하면, 이를 검토하여..."

## Step 1: Product Big Picture

```markdown
### 제품 개요

**[제품군] 소개:** [상위 제품의 한 줄 설명]

**제품 계층 구조:**
[전체 제품에서 이 기능의 위치를 트리 구조로 표시]

**[분석 대상 기능] 소개:** [한 줄 설명 + 주요 모드]

**누가, 왜 사용하는가:**
- [역할 A]는 [목적 A]를 위해 사용한다

**이 제품이 없다면:**
[제품이 해결하는 문제를 구체적으로]
```

**이 제품 개요에 Phase 2 Step 1.5의 도메인 워크플로우를 포함.** 제품 소개 직후에 "이 도메인은 어떻게 진행되나? (배경지식)"을 배치.

## Step 2: Pipeline Overview

Phase 2 Step 1.5의 **도메인 워크플로우**를 기반으로 **제품의 실제 사용 흐름**을 파이프라인으로 구성.

- 도메인 워크플로우 = 도메인 자체의 업무 프로세스 (배경지식, "제품 개요" 섹션)
- 제품 파이프라인 = 이 제품에서 실제로 수행하는 단계 (사용 흐름, "제품 사용 흐름" 섹션)

```markdown
### 전체 파이프라인

[Step 1] [단계명]
    [한 줄 설명]
        ↓
[Step 2] [단계명]
    [한 줄 설명]
        ↓
...
[Step N] [단계명]
    [한 줄 설명]

> 도메인 워크플로우("제품 개요" 섹션)의 [X]~[Y] 구간을 자동화합니다.
```

## Step 3: Step-by-Step Detail

각 파이프라인 단계 상세화:

```markdown
### Step N: [단계명]

**목적**: [이 단계가 왜 필요한지, 도메인 맥락에서 설명]

**사용자가 하는 일**: [구체적 설명]

**이 단계에서 다루는 항목** (해당 시):
| 항목 | 도메인적 의미 | 예시 (관통 예시) |
|------|-------------|-----------------|

**예시**: [관통 예시(running example)로 이 단계 설명]

**핵심 판단 기준** (해당 시): [사용자가 내려야 하는 핵심 판단]
```

**RULE: 항목 테이블은 도메인적 의미로 서술.**
- BAD: `| 가설 | textarea, required |` — UI 스펙
- GOOD: `| 가설 | 연구의 방향을 결정. 가설 없으면 데이터 수집 방법 결정 불가 | "SGLT-2 억제제 복용군은..." |`

**RULE: 각 Step은 도메인 워크플로우와 연결.**
```markdown
### Step 2: AI 연구설계 자동 생성
> 도메인 워크플로우 [2]~[6] — "가설부터 데이터 수집까지 자동화"
```
