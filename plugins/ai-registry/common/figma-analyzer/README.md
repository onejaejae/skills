# figma-analyzer

Figma 디자인 분석을 통해 제품과 도메인을 이해하는 스킬

## 설치

```bash
/plugin install figma-analyzer@ai-registry
```

## 사용법

### 트리거

- "figma 분석", "피그마 분석", "디자인 분석"
- "analyze figma", "figma analysis"
- "요구사항 추출", "플로우 정리", "도메인 파악"
- "이 디자인 봐줘", "피그마 봐줘"

### 핵심 원칙

UI 스펙이 아닌 **제품과 도메인**을 이해하는 것이 목표입니다.

- 도메인이 어떤 세계인지, 왜 이렇게 작동하는지 파악
- 제품의 big picture와 기능 간 관계 이해
- 개발자가 더 나은 의사결정을 내릴 수 있는 도메인 지식 구축

### 분석 단계

1. **Phase 0**: 분석 목표 명확화
2. **Phase 1**: 파일 구조 파악 (Orient)
3. **Phase 2**: 도메인 심층 분석 (Domain Deep Dive)
4. **Phase 3**: 제품 사용 흐름 (Product Pipeline)

### 지원 모드

| 모드 | 설명 | 예시 |
|------|------|------|
| Quick | 단일 화면, 빠른 개요 | "이 화면 하나만 봐줘" |
| Standard | 전체 분석 + 문서 작성 | "이 피그마 분석해줘" |
| Deep | 전체 분석 + 후속 질의 | "전체 가이드 정리해줘" |
