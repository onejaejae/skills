# Interview Plugin

사용자를 역인터뷰하여 상세하고 실행 가능한 스펙을 추출하는 스킬입니다.
수동적으로 요구사항을 받는 대신, AI가 체계적으로 질문하여 요구사항을 도출합니다.

## 주요 특징

- **코드베이스 기반 질문** — 코드를 먼저 스캔하여 이미 답이 있는 질문은 건너뜀
- **다중 라운드 심층 인터뷰** — AskUserQuestion으로 라운드당 2-3개 질문, 이전 답변 위에 심화
- **적응적 깊이 조절** — 구체적 작업은 1-2라운드, 복잡한 아키텍처는 4-6라운드
- **구조화된 스펙 산출물** — `specs/{topic-slug}.md`에 일관된 형식으로 작성
- **명시적 완료 기준** — "이해한 것 같으면 시작"이 아닌, 체크리스트 기반 종료 판단

## 인터뷰 카테고리

| 카테고리 | 집중 영역 |
|----------|-----------|
| **신규 기능** | 사용자 스토리, 수용 기준, 엣지 케이스, 기술 제약 |
| **리팩토링** | 현재 문제점, 목표 아키텍처, 마이그레이션 전략, 리스크 |
| **버그 분석** | 증상, 재현 절차, 환경, 기대 vs 실제 동작 |
| **아키텍처 설계** | 요구사항, 확장성, 트레이드오프, 연동 포인트 |

## 설치

```bash
/plugin install interview@ai-registry
```

## 사용법

```bash
/interview [주제]
```

### 트리거

- `/interview`, `interview me`, `인터뷰`
- `요구사항 추출`, `스펙 작성`, `요구사항 정리해줘`

## 워크플로우

1. **Context Discovery** — 코드베이스 스캔 + 주제 분류
2. **Deep Interview** — 다중 라운드 구조화 질문
3. **Completion Check** — 완료 기준 충족 확인
4. **Spec Output** — `specs/{topic-slug}.md` 작성
